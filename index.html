Developer Guide

Canton Network is a privacy-enabled interoperable blockchain network designed for institutional use. It connects independent applications into a single network while preserving data confidentiality through sub-transaction privacy.

Canton uses Daml as its smart contract language, enabling developers to model complex multi-party workflows with built-in authorization and privacy guarantees.

Read the Canton Network whitepaper

  • Sub-transaction privacy — participants only see the parts of a transaction they're involved in
  • $1.5T+ monthly volume — real transaction volume across the network
  • 600+ validators — securing the network
  • Featured App rewards — earn Canton Coin for building valuable applications
  • Composability — your app can interact with any other Canton application
  • Enterprise-grade — built for regulated industries

Get a Canton app running locally:

bash 
# Install the Daml SDK
curl -sSL https://get.daml.com/ | sh

# Create a new project
npx create-canton-app myapp
cd myapp

# Build your contracts
daml build

# Run tests
daml test

# Start the local sandbox
daml start

Your app is now running with a Daml sandbox, JSON API, and Navigator UI at localhost:7500.

Daml is a purpose-built smart contract language. Core concepts: templates (data and rules), choices (actions), signatories (authorization), and observers (visibility).

daml 
template Iou
  with
    issuer : Party
    owner : Party
    amount : Decimal
    currency : Text
  where
    signatory issuer
    observer owner

    choice Transfer : ContractId Iou
      with newOwner : Party
      controller owner
      do create this with owner = newOwner

    choice Settle : ()
      controller issuer
      do return ()

This IOU template demonstrates all four concepts: the template holds data, Transfer and Settle are choices, issuer is the signatory, and owner is an observer.

daml 
data EscrowState
  = AwaitingDeposit
  | Funded
  | Released
  | Disputed
  deriving (Eq, Show)

template Escrow
  with
    payer    : Party
    receiver : Party
    amount   : Decimal
    asset    : Text
    state    : EscrowState
    deadline : Time
  where
    signatory payer
    observer receiver

    ensure amount > 0.0

    choice Deposit : ContractId Escrow
      controller payer
      do
        assertMsg "Must be AwaitingDeposit" $
          state == AwaitingDeposit
        create this with state = Funded

    choice Release : ContractId Escrow
      controller receiver
      do
        assertMsg "Must be Funded" $ state == Funded
        create this with state = Released

    choice Dispute : ContractId Escrow
      controller receiver
      do
        assertMsg "Must be Funded" $ state == Funded
        create this with state = Disputed

    choice Refund : ContractId Escrow
      controller payer
      do
        now <- getTime
        assertMsg "Deadline has not passed" $
          now >= deadline
        assertMsg "Must be Funded" $ state == Funded
        create this with state = Released

Daml Script lets you test contracts by simulating multi-party workflows:

daml 
test_iou_transfer = script do
  alice <- allocateParty "Alice"
  bob <- allocateParty "Bob"

  iou <- submit alice do
    createCmd Iou with
      issuer = alice
      owner = alice
      amount = 100.0
      currency = "USD"

  submit alice do
    exerciseCmd iou Transfer with
      newOwner = bob

  return ()

Use canton-ci-templates to add CI/CD to your Daml project:

yaml 
# .github/workflows/ci.yml
name: Daml CI

on:
  push:
    branches: [main]
  pull_request:

jobs:
  build-and-test:
    uses: JohnLilic/canton-ci-templates/.github/workflows/daml-ci.yml@main
    with:
      daml-sdk-version: "2.9.0"
      run-tests: true
      upload-dar: true

The deployment path for Canton applications:

1
Local Sandbox — develop and test with daml start
2
DevNet — integration testing with other applications
3
TestNet — real network conditions and participants
4
MainNet — live on Canton Network

Each stage validates interoperability while increasing the scope of testing.

  • Canton Network rewards builders through the Featured Application program
  • 50% of new Canton Coin is allocated to application builders
  • Fee burn multipliers reward apps that generate network activity
  • Applications must demonstrate real utility and user adoption
  • Validator nodes earn additional rewards for securing the network

Canton Foundry is pursuing validator status to contribute directly to network infrastructure while building applications that qualify for Featured Application rewards.
